Инструменты для работы

Ниже описаны инструкции по работе с программами/сервисами.

Скраппер

Cервис, который позволяет нам скопировать верстку страницы со всем html/css/img/js файлами, для комфортной работы. Если в вашей задаче написано: взять скраппером, то речь идет об этом сервисе.

Инструкция проста: переходим по ссылке, копируем ссылку которую вы хотите скачать, скачиваете архив со всем необходимым.

Тестовый сервер(commander/сервер/тестовый)

Предназначен для заливки лендингов транзиток по ссылкам, с последующей передачей на тестирование. Для каждого верстальщика предусмотрен аккаунт, который регистрирует Тимлид отдела верстки.

Для загрузки страницы на сервер необходимо выполнить следующие действия:

  • необходимо зайти в коммандер
  • нажать на кнопку добавить лендинг(скрин).
  • Для корректного добавления обязательно нужно заполнить несколько полей:
    • Название лендинга(можно называть по названию продукта)
    • Ссылка на лендинг(писать свои инициалы(например Слобода Андрей - s.a.), название продукта, язык (указать как локализацию, например ru/en/de/el/it и т.д.), тип страницы(lp/tp(lp1,lp2/tp1/tp2)) + обязательно добавить в конце доменное имя rcktprft.ru, пример: sa.normalife.ru.lp.rcktprft.ru)
    • Архив необходимо загрузить в формате .zip, и только .zip
  • Также обязательно нужно выбрать, тип страницы(лендинг/транзитка)
  • Нажать кнопку добавить, дождаться деплоя, и открыть свою страницу кликом по доменному имени.

Ориентироваться, загрузилась ли страница, можно по колонке статус. Доступны следующие статусы: задеплоен(страница успешно загружена), деплой(страница еще не загрузилась, появляется чаще всего, когда возникла проблема), обновляется(страница еще не загрузилась).

Чтобы добавить цену на лендинг, необходимо перейти в редактирование лендинга(или указать при добавлении лендинга), в поле id оффера проставляем следующие цифры:

  • обычный оффер - 1 (цена 990/459)
  • рублевый оффер - 2 (цена 1 руб/1грн)
  • 99рублевый оффер- 3 (цена 99руб)
  • порционный оффер(147руб) - 4 (цена 147/67руб)

Так же от id оффера подтягиваются и доступные гео(страны и цены для них) для страницы. На тестовом сервере это фиксированый набор стран. На продакшине зависит от оффера. Наличие доступных стран по офферу можно посмотреть в СРА системе, или спросить у своего руководителя.

Для корректного отображения цены в поле "Размер скидки" указываем значение - 50 (50% скидка)

Примечание: Вывод страны работает таким образом - если вы заходите со страны, которая есть в СРА, и сервер определил ваше местоположение,то вам отображается текущий город/страна, если нет, с тем, которое установлено по умолчанию(на случай использования VPN могут возникать баги, когда определяется гео с погрешностью, выдает название города, который находится недалеко от вас, в пределах страны).

Так же есть поле поиска - сейчас работает только по ссылке - test-request.rcktprft.ru без дополнительных знаков(т.е. при копировании с браузера, необходимо убирать http и знаки //)

Production сервер(он же прод/продакшин)

Работает он по такой же схеме как и тестовый сервер. Только у вас права ограничены, и доступ предоставлен только для просмотра и скачивания архивов. На продакшин сервере создана учетная записть, которая позволяет верстальщику искать архивы по ссылкам/номеру в коммандере/названи/, и скачивать их для работы.

Task manager (JIRA)

Сейчас компания ведёт свою деятельность в Jira. У каждого верстальщика есть персональный аккаунт, который регистрируется по выходе на ИС. В Jira доступны следющие инструменты для контроля своих задач, доска Kanban, фильтры по задачам(текущие/ваши/отдела/завершенные и т.п.). Если вам поручают задачу, то должны поставить вас исполнителем в этой задаче. Если это не сделано, попоросите человека, который дал вам эту задачу, назначить вас. Если над задачей необходима работа нескольких сотрудников, они добавляются в рабочую группу, и имеют такие же права как и исполнитель.
